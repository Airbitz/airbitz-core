NAME

abc-cli - Airbitz command line tool

SYNOPSIS

abc-cli [ general_arguments ] command [ command_args ]

DESCRIPTION

The Airbitz CLI implements a decentralized, secure, peer-to-peer synchronized, and backed up bitcoin wallet.

The Airbitz wallet is a fully client-side wallet that creates, manages, and encrypts private keys, public keys, and transactions on the client device and uses server side assistance for backup and synchronization of encrypted data and for authenticating users. Airbitz users retain full control of their private keys at all times with neither Airbitz nor third parties having access to private keys or transaction data.

The abc-cli program is a command line tool for using the various Bitcoin functions of the ABC library from the shell.

GENERAL ARGUMENTS

Several commands require initial arguments that can be provided via command line or in the airbitz.conf file. Those arguments need to be set before the actual command.

-d <dir>

the working directory where the wallet information are stored.

-u <username>

the account username.

-p <password>

the account password.

-w <wallet>

the wallets id.

COMMAND SUMMARY

The abc-cli program provides a rich variety of commands (command in the SYNOPSIS above), each of which often has a wealth arguments (command_args in the SYNOPSIS).

GENERAL COMMANDS

check-password <pass>

Checks the password strength.

Requires a working directory.

general-update

Downloads general info (Electrum / libbitcoin Servers, Sync Servers, mining fees etc.) from the server if the local file is out of date.

Requires a working directory.

get-question-choices

Returns the recovery questions choices.

Requires a working directory.

list-accounts

List all accounts in the working directory.

Requires a working directory.

pin-login

Login using username + PIN instead of password.

Requires a working directory and username.

watcher

Starts a endless watcher-loop that looks for new incoming transactions.

Requires a working directory, username, password and wallet.

version

Returns the ABC Version.

ACCOUNT COMMANDS

Laccount-available

Checks if the username is available.

Requires a working directory and username.

account-create <user> <pass>

Creates a new account. Default PIN set is 1234.

Requires a working directory.

account-decrypt <filename>

The filename is account-relative.

Requires a working directory, username and password.

account-encrypt <filename>

The filename is account-relative.

Requires a working directory, username and password.

change-password <new-pass>

Requires a working directory, username and password.

change-password-recovery <ra> <new-pass>

Changes the password to <new-pass> with the recovery answers.

Requires a working directory and username.

check-recovery-answers <answers>

Requires a working directory and username.

data-sync

Syncs the account and all its wallets.

Requires a working directory, username and password.

get-question

Returns the recovery questions.

Requires a working directory and username.

get-settings

Returns the accounts settings (first name, last name, nickname, language, etc.)

Requires a working directory, username and password.

pin-login-setup

Sets up the data for a pin-based login, both on disk and on the server.

Requires a working directory, username and password.

recovery-reminder-set <n>

Requires a working directory, username and password.

set-nickname <name>

Set the account nickname.

Requires a working directory, username and password.

sign-in

Sign-in an account.

Requires a working directory, username and password.

upload-logs

Uploads the logs.

Requires a working directory, username and password.

SPEND COMMANDS

spend-uri <uri>

Sends Bitcoin using a payment URI.

Requires a working directory, username, password and wallet.

spend-internal <address> <amount>

Requires a working directory, username, password and wallet.

spend-transfer <wallet-dest> <amount>

Transfers Bitcoins from one wallet to another.

Requires a working directory, username, password and wallet.

ADDRESS COMMANDS

address-allocate <count>

Returns new receive addresses and mark them as used.

Requires a working directory, username, password and wallet.

address-list

List all addresses used by the wallet.

Requires a working directory, username, password and wallet.

address-search <addr> <start> <end>

Searches if the address exists between start and end.

Requires a working directory, username, password and wallet.

CATEGORIES COMMANDS

category-list

List all categories.

Requires a working directory, username and password.

category-add <category>

Add a new category.

Requires a working directory, username and password.

category-remove <category>

Remove a category.

Requires a working directory, username and password.

EXCHANGE COMMANDS

exchange-fetch

Fetches all exchange rates.

Requires a working directory.

exchange-update <currency>

Returns the current Bitcoin price in <currency>

Requires a working directory, username and password.

exchange-validate <currency>

Validates that all currencies have sources.

Requires a working directory.

OTP COMMANDS

otp-key-get

Returns the OPT key.

Requires a working directory and username.

otp-key-set <key>

Set the OPT key.

Requires a working directory and username.

otp-key-remove

Remove the OPT key.

Requires a working directory and username.

otp-auth-get

Returns the OPT Auth timeout.

Requires a working directory, username and password.

otp-auth-set <timeout-sec>

Sets the OPT Auth timeout.

Requires a working directory, username and password.

otp-auth-remove

Remove the OPT key.

Requires a working directory, username and password.

otp-reset-get

Returns the reset status for all accounts.

Requires a working directory

otp-reset-remove

Cancels an OTP reset timer.

Requires a working directory, username and password.

BitID COMMANDS

bitid-login <uri>

Performs a BitID login to the specified URI.

Requires a working directory, username and password.

bitid-sign <uri> <message> [<index>]

Signs a message.

Requires a working directory, username and password.

PLUGIN COMMANDS

plugin-get <plugin> <key>

Retrieves an item from the plugin key/value store.

Requires a working directory, username and password.

plugin-set <plugin> <key> <value>

Saves an item to the plugin key/value store.

Requires a working directory, username and password.

plugin-remove <plugin> <key>

Deletes an item from the plugin key/value store.

Requires a working directory, username and password.

plugin-clear <plugin>

Removes the entire key/value store for a particular plugin.

Requires a working directory, username and password.
